# История изменений функционала (ELS AlexA V2.1)

## Файл

`02_Документация/CHANGELOG.md`

Новые записи — **сверху**, под заголовком `# История изменений`.

## Шаблон записи

```markdown
## YYYY-MM-DD

### Краткий заголовок (1 строка)

- **Промпт:** суть запроса пользователя (1–2 предложения, без воды).
- **Результат:** что изменилось в поведении / что сделано (1–3 предложения).
- **Область:** модули или темы (например: backlash, LCD, joy, limits).
```

Дата — из контекста сессии (`Today's date`), формат `YYYY-MM-DD`.

## Правила содержания

- Пиши по-русски, кратко.
- **Промпт** — сжатая формулировка задачи, не дословная длинная цитата.
- **Результат** — эффект для системы/оператора, не перечень каждого изменённого файла.
- Одна логическая задача → одна запись (или один блок `###` под общей датой).
- Несколько задач в один день — несколько `###` под одной `## YYYY-MM-DD`.
- Не логируй отказ от правок, только размышления и Ask-mode без изменений кода/доков продукта.

## Чеклист агента

1. Правки функционала завершены и приняты в этой сессии.
2. Открыть / создать `02_Документация/CHANGELOG.md`.
3. Вставить запись по шаблону в начало истории.
4. Не создавать параллельные changelog-файлы.
