# Подхват резьбы через Divider — COU как ΔZ (перспектива)

> **Статус: не реализовано.** Документ фиксирует целевое поведение на этап
> внедрения режима нарезания резьбы (`mode_thread`). Текущий divider показывает
> в COU обороты:фазу шпинделя — см. `els-divider`.

## Идея

Шпиндель и ось Z можно перемещать **произвольно и независимо**, но контроллер
**непрерывно отслеживает**:

- полный счёт шпинделя (целые обороты + фаза, знаковый `spindle_get_count()`);
- фактическую позицию Z (`motion_get_pos_steps(AXIS_Z)`).

При заданном **шаге резьбы P** (мм/об) и активном **отслеживании подхвата** в любой
момент можно вычислить, на сколько нужно сдвинуть Z, чтобы попасть в шаг нитки.
Эта величина отображается в **строке 2 LCD** вместо (или в режиме) COU.

Оператор:

1. Включает нарезание резьбы, задаёт шаг **P**.
2. Произвольно крутит шпиндель, фиксирует угол (как R→0 в divider).
3. Смотрит **ΔZ** на строке COU, смещает Z (джойстик/РГИ) на эту дельту.
4. Включает синхронную подачу — движение идёт в шаге резьбы.

## Связь с делительной головкой

| Элемент divider (сейчас) | Подхват резьбы (перспектива) |
|--------------------------|------------------------------|
| **R** — угол до цели деления | **R** — без изменений (деление окружности) |
| **COU** — обороты:фаза шпинделя | **COU** — **ΔZ** корректировки оси Z |
| Select — сброс опорного угла шпинделя | Select — сброс пары `(C_ref, Z_ref)` |
| Шпиндель вручную | Шпиндель и Z вручную, связь только вычисляемая |

Divider остаётся режимом деления; опция подхвата — **надстройка**, активируемая при
включении функции нарезания резьбы и задании **P**.

## Условие стыковки (виртуальная гайка)

Опорная точка после Select (или отдельной команды «зафиксировать стыковку»):

```
C_ref — spindle_get_count() в момент фиксации
Z_ref — motion_get_pos_steps(AXIS_Z) в тот же момент
P     — шаг резьбы, мм/об (из mode_thread / меню)
```

В любой момент:

```
TICKS_PER_REV = SPINDLE_ENCODER_PPR   /* INT0 Rising на B: ×1 к PPR; см. els-divider */
C             = spindle_get_count()
Z             = motion_get_pos_steps(AXIS_Z)

Z_target_mm   = Z_ref_mm + (C - C_ref) * P / TICKS_PER_REV
ΔZ_mm         = Z_target_mm - Z_current_mm
```

В шагах (для jog/planner):

```
ΔZ_steps = (C - C_ref) * P_steps_per_rev / TICKS_PER_REV - (Z - Z_ref)
```

где `P_steps_per_rev = P_mm * STEPS_PER_MM_Z`.
Не использовать `PPR * 2` — устарело после смены Any Change → Rising.

**ΔZ = 0** — ось Z согласована с шпинделем и шагом P; можно включать sync.

### Почему нужны обороты + фаза

Учитывается **полный** знаковый счёт `C`, не только угол 0…360°. Иначе ошибка
кратна **P** за каждый потерянный полный оборот.

## LCD (строка 2 при активном подхвате)

### Сейчас (divider, реализовано)

```
COU ±NNNNNNN:PPPP    /* целые обороты : фаза (4 цифры) */
```

### Целевое (подхват резьбы активен)

```
COU ±ZZZZZ.zz        /* или COU ±ZZ.zzz — дельта Z в единицах CrdU */
```

| Поле | Смысл |
|------|--------|
| Знак | направление смещения Z для стыковки (+ вперёд / − назад) |
| Величина | **ΔZ** до попадания в шаг нитки |
| Единицы | как координаты (мм / дюймы / шаги) — флаг CrdU в конце строки при необходимости |

Примеры (мм):

```
COU +0.125           /* сдвинуть Z на +0.125 мм */
COU -1.400           /* сдвинуть Z на −1.4 мм */
COU +0.00            /* в шаге — можно включать sync */
```

Строки 0, 1, 3 divider без изменений (угол, A B R, координаты).

## Активация опции

| Событие | Поведение |
|---------|-----------|
| Режим divider (FSM 6), резьба **не** активна | COU = обороты:фаза (как сейчас) |
| Активирована **нарезание резьбы** + задан **P** | COU = ΔZ; ведётся `(C_ref, Z_ref)` |
| Select (короткое) | Обновить `C_ref`, `Z_ref` (стыковка в текущей точке) |
| ΔZ → 0 (в пределах порога) | Длинный бип (аналог R→0 в divider) |
| Уход ΔZ с нуля | Два коротких бипа |

Порог нуля ΔZ — согласовать с разрешением энкодера и CrdU (ориентир ±0.01 мм или 1 шаг).

## Алгоритм оператора

1. Включить divider, активировать нарезание резьбы, задать **P**.
2. **Select** — зафиксировать опору `(C_ref, Z_ref)` (или после первого прохода резьбы).
3. Произвольно повернуть шпиндель.
4. По **COU (ΔZ)** сместить Z до нуля (или близко).
5. Включить синхронную подачу по Z с тем же **P**.
6. Повторять п. 3–5 при переподхвате.

## Ограничения (обязательно учитывать при реализации)

1. **Люфт Z** (`els-backlash`) — физическая позиция может отставать от расчётной
   на величину backlash; ΔZ показывает *командную* стыковку.
2. **Пропуск шагов / jog без учёта** — связь нарушается, нужен новый Select.
3. **P должен совпадать** с подачей sync/thread — иначе дрейф по длине.
4. **Знак направления** шпинделя и резьбы (LH/RH) — единая конвенция в формуле.
5. **Подхват существующей нарезки** — если `(C_ref, Z_ref)` не от этой нитки,
   ΔZ обнуляется в пределах одного шага P, но может потребоваться визуальный подбор
   в окне ±P.
6. Форматирование и расчёт ΔZ — **только в loop**, не в ISR.

## Чеклист (при реализации)

```
- [ ] Флаг: thread_align активен только при mode_thread + задан P
- [ ] COU строка 2: ΔZ в единицах CrdU, знак направления
- [ ] Select: latch C_ref + Z_ref (не только spindle)
- [ ] Формула: полный счёт C, не только фаза
- [ ] Бип: ΔZ→0 длинный, уход с 0 — двойной короткий
- [ ] Без sync/planner до ΔZ≈0 (оператор вручную)
- [ ] Откат к COU обороты:фаза при выключении thread_align
- [ ] Логи DBG: latch, ΔZ, порог нуля
```
