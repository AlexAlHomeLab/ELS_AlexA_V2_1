Отладка ELS AlexA V2.1

Принципы отладки

1. Телеметрия для AI
- Структурированный формат
- Понятен человеку
- Парсится автоматически
- Достаточно контекста

2. Условная компиляция
DEBUG_ENABLED = 1 для разработки
DEBUG_ENABLED = 0 для production

3. Уровни отладки
- ERROR (всегда)
- WARNING
- INFO
- VERBOSE

Формат телеметрии

Структура сообщения:
[timestamp] [LEVEL] [MODULE] [COMPONENT] message

Пример:
[2026-07-09 11:30:45.123] [INFO] [MOTION] [STEPPER] Step generated: axis=X, pos=12345

Что логировать

Всегда:
- Критические ошибки
- Срабатывание защит
- Изменение режимов
- Движения

По запросу:
- Параметры движений
- Состояние кнопок
- Значения энкодеров
- Координаты

Для отладки:
- Вход/выход из функций
- Временные значения
- Состояния FSM

Каналы отладки

1. Serial - основной канал, человекочитаемый
2. Serial Config CLI - настройка параметров (GRBL-стиль, коды $)
3. Telemetry buffer - структурированный, для автоматического анализа
4. LED индикация - быстрая визуальная проверка

Serial Config CLI — GRBL-стиль (115200)

Команды:
- `$$` — все настройки (`$n=value`)
- `$n` — одна настройка
- `$n=value` — запись в EEPROM
- `$I` — версия прошивки
- `$H` — справка по кодам
- `?` — статус (MPos, FSM, режим, RPM)

Ответ: `ok` или `error: ...`

Таблица кодов:
- `$0-$1` — async feed min/max (mm/min)
- `$2-$3` — sync feed min/max (mm/rev)
- `$10-$15` — Z: steps, ustep, pitch, max, rapid, accel
- `$16` — Z steps/mm (только чтение)
- `$20-$25` — X: steps, ustep, pitch, max, rapid, accel
- `$26` — X steps/mm (только чтение)
- `$30` — spindle PPR, `$31` — buzzer (0/1)

Примеры:
```
$$
$1=500
$12=1.60
$14=2000
$23=400
$I
?
```

Анализ логов

Инструменты:
- analyze_log.py - парсинг логов
- Визуализация данных

Что анализировать:
1. Временные метки (задержки)
2. Последовательность событий
3. Ошибки и предупреждения
4. Производительность

Отключение отладки

Production: DEBUG_ENABLED = 0
Development: DEBUG_ENABLED = 1

Правила написания дебаг-кода

Не замедлять работу:
- Отладочные проверки минимальны
- Использовать макросы для быстрого пропуска
- Не блокировать прерывания

Не засорять память:
- Ограниченный буфер телеметрии
- Обрезка длинных строк
- Циклическая запись
