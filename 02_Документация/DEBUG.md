Отладка ELS AlexA V2.1

Принципы отладки

1. Телеметрия для AI
- Структурированный формат
- Понятен человеку
- Парсится автоматически
- Достаточно контекста

2. Условная компиляция
DEBUG_ENABLED = 1 для разработки
DEBUG_ENABLED = 0 для production

3. Уровни отладки
- ERROR (всегда)
- WARNING
- INFO
- VERBOSE

Формат телеметрии

Структура сообщения:
[timestamp] [LEVEL] [MODULE] [COMPONENT] message

Пример:
[2026-07-09 11:30:45.123] [INFO] [MOTION] [STEPPER] Step generated: axis=X, pos=12345

Что логировать

Всегда:
- Критические ошибки
- Срабатывание защит
- Изменение режимов
- Движения

По запросу:
- Параметры движений
- Состояние кнопок
- Значения энкодеров
- Координаты

Для отладки:
- Вход/выход из функций
- Временные значения
- Состояния FSM

Каналы отладки

1. Serial - основной канал, человекочитаемый
2. Telemetry buffer - структурированный, для автоматического анализа
3. LED индикация - быстрая визуальная проверка

Анализ логов

Инструменты:
- analyze_log.py - парсинг логов
- Визуализация данных

Что анализировать:
1. Временные метки (задержки)
2. Последовательность событий
3. Ошибки и предупреждения
4. Производительность

Отключение отладки

Production: DEBUG_ENABLED = 0
Development: DEBUG_ENABLED = 1

Правила написания дебаг-кода

Не замедлять работу:
- Отладочные проверки минимальны
- Использовать макросы для быстрого пропуска
- Не блокировать прерывания

Не засорять память:
- Ограниченный буфер телеметрии
- Обрезка длинных строк
- Циклическая запись
