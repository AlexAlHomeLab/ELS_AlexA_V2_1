# История изменений

Журнал изменений **функционала** прошивки ELS AlexA V2.1.  
Формат и правила: `.cursor/skills/els-changelog/SKILL.md`.

## 2026-07-20

### Zero/cooldown: не сбрасывать mpg_dir_lock

- **Промпт:** реверс после сброса координаты без паузы в 4 шага (`dir` сразу + BL).
- **Результат:** `sc_cooldown_arm` больше не обнуляет `mpg_dir_lock` (zero → cool раньше затирал restore); после zero обратные тики снова `rev ign` → смена на 4-м.
- **Область:** motion_jog, MPG, zero, set-coord cooldown.

### Zero оси: сохранить mpg_dir_lock (rev ign N)

- **Промпт:** пропала задержка в 4 тика РГИ перед сменой направления и выборкой люфта после zero.
- **Результат:** zero больше не сбрасывает `mpg_dir_lock` — обратные тики снова идут через `rev ign` 1…N, смена DIR/BL на (N+1)-м тике.
- **Область:** motion_jog, MPG, zero.

### Zero оси: без sync люфта (ложный last_dir)

- **Промпт:** после сброса координаты в 0 медленный РГИ в любую сторону не включает компенсацию люфта.
- **Результат:** `motion_jog_zero_axis` больше не вызывает `backlash_sync(REF_DIR)` — zero только координата; `last_dir` сохраняется, выборка при реальной смене DIR.
- **Область:** motion_jog, backlash, zero/LimRe.

### РГИ: idle stop через halt (без DECEL после паузы)

- **Промпт:** рывок после остановки через паузу — в логе `idle stop` затем `DECEL rem45`.
- **Результат:** по `MPG_IDLE_MS` вызывается `planner_jog_halt` вместо `jog_stop`/`jog_release` — нет доката торможения после паузы.
- **Область:** motion_jog, MPG.

### РГИ: sync cmd при обгоне + idle stop на pos

- **Промпт:** медленный РГИ не едет (hold tgt), быстрый едет; после паузы иногда ещё ход (`idle catch`).
- **Результат:** если pos обогнал `mpg_cmd` — sync до тиков/в commit (тики снова двигают ось); по `MPG_IDLE_MS` стоп на pos без догона cmd (нет хода после паузы).
- **Область:** motion_jog, MPG.

### РГИ: lead pad только при обгоне мотора (нет рывка после BL)

- **Промпт:** после rev stop / BL START снова рывок — в логе `ext -2` (cmd−128) и idle catch.
- **Результат:** lead за `mpg_cmd` ставится только если РГИ уже обгоняет позицию (`gap > lead`); после люфта и на x1 цель = `mpg_cmd`, без прыжка на MAX_LEAD.
- **Область:** motion_jog, MPG.

### РГИ: lead только после сдвига pos (после BL)

- **Промпт:** после rev stop всё ещё `ext cmd+128` сразу за `BL START` — рывок.
- **Результат:** lead pad разрешён только когда позиция сдвинулась с холодного старта (выборка люфта закончена); гонка «тиk до arm rem» устранена. В логе не должно быть `ext` во время/сразу после `BL START`.
- **Область:** motion_jog, MPG, backlash.

### РГИ: без lead во время выборки люфта

- **Промпт:** после rev stop всё ещё рывок (`BL START` + `ext` к cmd+lead).
- **Результат:** lead pad не навешивается, пока `backlash_pending`; idle catch ждёт окончания выборки — нет рывка к lead-цели после люфта.
- **Область:** motion_jog, MPG, backlash.

### РГИ: смена оси без halt/рывка

- **Промпт:** после смены оси и возврата — рывок; переключение не должно двигать оси и наследовать сессию.
- **Результат:** смена селектора оси только сбрасывает сессию РГИ (dir_lock/active/APPROACH, discard тиков, sync cmd новой оси); `planner_jog_halt` убран; пустой cruise на цели снимается без хода; чужой едущий target не затирается.
- **Область:** motion_jog, MPG.

### РГИ: rev stop без доката в старом направлении

- **Промпт:** после хода и смены направления на 4-м клике короткий ход «назад», затем выборка люфта и рывок.
- **Результат:** при реверсе >N импульсов — `planner_jog_halt` (мгновенный стоп) вместо `planner_jog_stop`/jog_release; нет доката в старом DIR перед новым ходом и люфтом.
- **Область:** motion_jog, MPG.

### Set-coord: изоляция MPG (busy + quiet cooldown)

- **Промпт:** при отпускании L/R/U/D и при нажатии без вращения РГИ бывают импульсы MPG — изоляция неполная.
- **Результат:** MPG блокируется при любой нажатой L/R/U/D (дребезг ≠ release); commit только когда все кнопки отпущены; после release — окно тишины энкодера 300 мс (тики продлевают).
- **Область:** ui_buttons, motion_jog, set-coord.

### Set-coord: полярность X, wrap .999, cooldown после release

- **Промпт:** инверсия знака X; дробная 999→000; дёрганья моторов при отпускании и при zero оси.
- **Результат:** знак тика X в set-coord инвертирован; правка дробной через val1000 (999+1→.000 с переносом); после release/zero — 200 мс cooldown с discard тиков РГИ (нет старта MPG); zero — halt + sync cmd + cooldown.
- **Область:** motion_jog, set-coord.

### Set-coord: без хода моторов и без спама pos

- **Промпт:** спам `SC pos`; при кнопке+РГИ крутятся моторы и прыгают другие поля LCD.
- **Результат:** set-coord с первого нажатия L/R/U/D (не ждать 600 мс) — MPG сразу halt/discard; правка только превью оси; убран VERBOSE `SC pos` каждый тик.
- **Область:** ui_buttons, motion_jog, set-coord.

### Set-coord: детект по raw Port F + halt MPG

- **Промпт:** HOLD L/R/U/D есть, set-coord не работает (идёт MPG rev ign).
- **Результат:** удержание ≥600 мс определяется по `PINF` (не EncButton.pressing); INFO `SC hold`/`arm`/`halt mpg`; активный MPG останавливается при входе в set-coord; тики РГИ правят превью.
- **Область:** ui_buttons, motion_jog, set-coord.

### Set-coord: диагностика блокировок + LCD превью

- **Промпт:** HOLD L/R/U/D есть, координат и отладки SC нет.
- **Результат:** set-coord вызывается до axis_arm/MPG-блоков; INFO `SC blk state|startup|MODE_OFF|golim`; arm с осью/шагом и первый `tick`; dirty LCD учитывает превью (иначе экран не обновлялся при правке без хода).
- **Область:** motion_jog, fsm_manager, LCD, set-coord.

### Отладка: без спама Serial (INFO по умолчанию)

- **Промпт:** проверить, чтобы отладка не спамила.
- **Результат:** `DEBUG_LEVEL` по умолчанию INFO (не VERBOSE); `F pot/rapid` — гистерезис ≥5 мм/мин; `coast`/`idle catch` — один VERBOSE на фазу сессии.
- **Область:** config, motion_jog, debug.

### РГИ: установка координат L/R/U/D + изоляция hand_pos

- **Промпт:** проверить РГИ по скилу, реализовать недостающее, исправить ошибки.
- **Результат:** hold Left/Right (X целая/дробная) или Up/Down (Z) + РГИ на главном экране правит превью координаты без хода; отпускание — commit (DDS, mpg_cmd, rebase лимитов), без хвоста в сессии MPG. Сброс hand_pos на фронте Rapid APPROACH убран (только joy). Комментарии селектора масштаба приведены к D15=x1 / D14=0.01.
- **Область:** motion_jog, LCD, MPG, set-coord.

### РГИ: полная отладка Serial (фронты A–I MPG)

- **Промпт:** восполнить пробелы DBG для полного контроля работы РГИ.
- **Результат:** INFO на блоках, dir/реверс, idle, lim, F-режиме, APPROACH (в т.ч. arm sync / at tgt), halt/axis/hand rst; VERBOSE — ticks/cmd/hand, coast, batch. Без спама каждый тик.
- **Область:** motion_jog, MPG debug.

### Rapid за лимитом: стоп без автовозврата

- **Промпт:** после проезда лимита на Rapid и отпускания джойстика суппорт разворачивался к Lim* (лог DIR/BL START).
- **Результат:** в `dds_motion_jog_release` clamp цели торможения не меняет знак движения — за лимитом дотормаживание по курсу, без подтягивания к лимиту; внутри зоны clamp к Lim* как раньше.
- **Область:** stepper_gen, joy, limits.

### Джойстик: расширенный DBG для сценариев A–I

- **Промпт:** максимум отладочных сообщений по всем режимам джойстика (подача, rapid, go_lim, latch, блокировки).
- **Результат:** INFO на фронтах rapid/скорости/оси, отказах latch/go_lim, blk lim, H→I (`stop_joy`), mode/sub вне MANUAL; VERBOSE — runway/clamp/`spd0 fail`. Без спама каждый тик.
- **Область:** motion_jog, ui_buttons, planner, fsm_manager, joy debug.

### Выборка люфта: без рывка на разгоне (РГИ/джойстик)

- **Промпт:** дёрганья на выборке люфта (лог DIR/BL START, CRUISE F1200, затем рывок).
- **Результат:** шаги выборки идут на скорости текущего профиля (`current_rate`); `motion_boost_backlash_rates` поднимает increment только до BlMn, а не до номинала arm; при смене DIR в ISR — `motion_apply_rates`.
- **Область:** stepper_gen, backlash.

### Обнуление оси (длинное нажатие лимита): без ложной выборки люфта

- **Промпт:** после сброса оси в 0 длинным нажатием лимита включается выборка люфта.
- **Результат:** при `motion_jog_zero_axis` — halt jog/MPG, сброс `mpg_cmd`/`hand_pos` по оси, `backlash_sync_axis`; coast к старой цели РГИ больше не запускает движение и люфт.
- **Область:** limits, motion_jog, backlash.

### JOY и РГИ: изоляция осей в planner

- **Промпт:** убрать дёрганья при одновременной подаче джойстиком и РГИ; источники должны быть изолированы.
- **Результат:** «чужая» ось в `planner_exec_jog` берёт `dds_get_target`, а не `position` — retarget РГИ не сбрасывает runway джойстика и наоборот; то же для APPROACH-подвода РГИ.
- **Область:** motion_jog, joy, MPG.

### РГИ: без рывка после выборки люфта при смене направления

- **Промпт:** после поворота РГИ в новую сторону — выборка люфта, затем короткий рывок.
- **Результат:** в jog cruise шаги bl крутят профиль разгона (current_rate не замирает); arm люфта для РГИ — та же runtime-подача, что и ход (`feed_mm_min` сегмента), не BlSp.
- **Область:** stepper_gen, MPG, backlash.

### Джойстик: скорость выборки люфта = подача

- **Промпт:** выборка люфта при jog заметно медленнее установленной подачи.
- **Результат:** arm люфта для jog — `runtime_speed` (как пот, без потолка BlSp); в cruise — nominal, не `current_rate` разгона; после arm и по окончании rem — обновление DDS increment (`boost` / `apply_rates`).
- **Область:** stepper_gen, backlash, joy.

### Джойстик: торможение при отпускании (Jdec, профиль)

- **Промпт:** движение джойстиком — люфт, разгон, подача с пота; при отпускании торможение с настраиваемым ускорением; треугольник/трапеция от момента отпускания; rapid и go_lim без регрессий.
- **Результат:** `dds_motion_jog_release` переводит cruise в конечный профиль торможения (от текущей скорости, дистанция max(Jdec, физика), cap 200 шагов); цель ставится вперёд по ходу на время decel; при старте JOY cruise учитывается pending люфт в длине разгона.
- **Область:** stepper_gen, joy, motion profile, Jdec.

### РГИ после джойстика: без рывка разгона

- **Промпт:** убрать рывок при первом тике РГИ после остановки джойстика; РГИ изолирован от джойстика, люфт только по направлению.
- **Результат:** lead РГИ только при живом cruise; jog-cruise для РГИ восстановлен (скорость из `config_mpg`, не от пота); разгон РГИ не длиннее хода; arm люфта при РГИ — по BlSp/BlMn, не от селектора шага.
- **Область:** motion_jog, planner, stepper_gen, MPG.

## 2026-07-18

### РГИ Z: вправо +, влево −

- **Промпт:** при вращении РГИ вправо координата Z должна расти, влево — уменьшаться.
- **Результат:** включён `MPG_AXIS_Z_INVERT` — знак тика Z инвертируется в `mpg_adjust_tick_sign`.
- **Область:** MPG, config_mpg, motion_jog.

### GO_LIM: стоп джойстиком без автостарта подачи

- **Промпт:** остановка go_lim/latch джойстиком должна только останавливать; повторный старт — после отпускания и нового включения.
- **Результат:** `go_lim_stop_joy` держится, пока джойстик удерживается; jog после стопа лимита не стартует без отпускания.
- **Область:** motion_jog, joy, go_lim/latch.

## 2026-07-17

### APPROACH: нет хвоста мелких шагов после подвода

- **Промпт:** после остановки энкодера и отпускания Rapid идёт подвод, затем ещё мелкие очень медленные шаги.
- **Результат:** подвод на cruise без хвоста торможения; по прибытии на цель — halt без coast/lead/bl_drain; энкодер на время подвода игнорируется.
- **Область:** motion_jog, MPG APPROACH.

### Обнуление оси: без sync люфта

- **Промпт:** при обнулении оси не должна включаться компенсация люфта.
- **Результат:** `motion_jog_zero_axis` больше не вызывает `backlash_sync_axis` — last_dir/synced сохраняются, ложной выборки после zero нет.
- **Область:** motion_jog, backlash, limits hold.

### Latch: стоп на ближайшем лимите по направлению

- **Промпт:** при возврате на лимит проезжали первый по направлению.
- **Результат:** `limits_ui_go_target_dir` выбирает ближайший активный лимит впереди по знаку джойстика, а не дальний край пары.
- **Область:** limits, joy latch.

### Hold лимита: zero без joy, latch при joy

- **Промпт:** сброс оси только длинным нажатием лимита без джойстика; при включённом джойстике — возврат к лимиту по направлению.
- **Результат:** hold любой Lim* без joy обнуляет ось кнопки; при joy — latch к активному лимиту в направлении джойстика (zero не выполняется).
- **Область:** ui_buttons, limits, joy latch.

### APPROACH только для РГИ, не для джойстика

- **Промпт:** APPROACH не должен влиять на джойстик — только на РГИ.
- **Результат:** arm APPROACH при Rapid ↑ только если джойстик не в подаче; при появлении JOY без тиков РГИ arm снимается. Отпускание Rapid после JOY не запускает подвод.
- **Область:** motion_jog, MPG/Rapid, joy.

### JOY/Rapid: нет автохода после отпускания за лимитом

- **Промпт:** после выхода за лимит на rapid отпуск джойстика, затем Rapid — самопроизвольное движение; нужен только стоп, возврат на лимит — джойстик+кнопка лимита.
- **Результат:** отпускание Rapid без тиков РГИ больше не запускает подвод к лимиту; `mpg_cmd` синхронизируется с позицией. Подвод APPROACH — только после вращения РГИ; возврат на лимит — только go_lim.
- **Область:** motion_jog, joy, Rapid, limits.

### РГИ APPROACH: шаг с селектора, не 0.1 от Rapid

- **Промпт:** при точном подводе включается 0.1 мм/тик, нужен масштаб селектора.
- **Результат:** в `MPG_RAPID_MODE_APPROACH` Rapid больше не переключает цену импульса на 0.1 мм — шаг как у селектора (x1 / 0.01). Режим 0.1 от Rapid остаётся только в LIVE.
- **Область:** motion_jog, MPG/Rapid.

### Rapid: нет отката к arm-pos (APPROACH)

- **Промпт:** после ухода за лимит Z на rapid отпуск джойстика и Rapid — лишний ход.
- **Результат:** при отпускании Rapid без тиков РГИ `mpg_cmd` синхронизируется с текущей позицией; подвод только при тиках РГИ; при удержании джойстика подвод не стартует. (Автовозврат на лимит убран — см. запись «JOY/Rapid: нет автохода…».)
- **Область:** motion_jog, MPG/Rapid.

### Backlash: рывок после выборки при смене направления

- **Промпт:** при смене направления после выборки люфта — рывок.
- **Результат:** скорость выборки при arm больше не режется потолком BlSp; идёт с подачей движения (не ниже BlMn). Убран скачок BlSp→подача в момент окончания rem.
- **Область:** backlash.

### JOY/STEP: lean ISR cruise + период 40 мкс

- **Промпт:** рывки на высокой скорости остались (LCD уже выкл).
- **Результат:** отдельный короткий ISR для jog-cruise (DDS+PORTL, без profile/backlash/fn-ptr); период Timer1 40 мкс (~25 кГц) с запасом; runway joy заполняется до 16000 шагов.
- **Область:** stepper_gen, config, motion_jog.

### JOY/STEP: рывки на высоких скоростях (ISR + runway)

- **Промпт:** на малых оборотах рывки пропали, на высоких остались; дисплей отключён.
- **Результат:** STEP/DIR через прямой PORTL; в cruise jog профиль — только счётчик шагов; rates на разгоне раз в 8 шагов; retarget джойстика только при остатке цели меньше 8192 шагов (не каждые 20 мс).
- **Область:** hal_pins, stepper_gen, motion_jog.

### JOY/STEP: убраны равномерные дыры в импульсах (рывки)

- **Промпт:** скорость появилась, но импульсы равноинтервально пропадают, рывки.
- **Результат:** cruise-retarget больше не держит `cli()` на float/`motion_apply_rates` каждые 20 мс; DIR/BL только при смене направления; runway цели до 16384 шагов, наращивание от текущей цели.
- **Область:** stepper_gen, planner, motion_jog, joy.

### STEP: частота ~4 кГц → потолок ISR (~30 кГц)

- **Промпт:** разобраться и исправить частоту импульсов STEP ~4.184 кГц при F2000.
- **Результат:** в cruise больше не вызывается `motion_apply_rates` на каждый шаг (uint64-div раздувал ISR до ~240 мкс); DDS capped по частоте Timer1; DIR не пишется каждый тик. Ожидаемо ~30 кГц на крейсере (потолок ISR).
- **Область:** stepper_gen, motion/ISR.

### JOY: убран реверс из‑за конфликта с РГИ

- **Промпт:** по логу Serial исправить джойстик (осцилляция DIR/BL при удержании).
- **Результат:** при JOY на оси РГИ сессия MPG сбрасывается; idle coast больше не тянет цель к старому `mpg_cmd` против джойстика.
- **Область:** motion_jog, joy, MPG/РГИ.

### РГИ: убран runaway lead (крутило в одну сторону)

- **Промпт:** после lead ось всегда едет в одну сторону, рывки остались.
- **Результат:** lead = `mpg_cmd±pad` (не `pos+lead` каждый poll); `mpg_cmd` lead’ом не переписывается; реверс >N → halt по ТЗ.
- **Область:** motion_jog, MPG/РГИ.

### РГИ: lead как у JOY — непрерывный STEP

- **Промпт:** проверить всю цепочку исполнения; рывки не уходят, подозрение на генерацию импульсов.
- **Результат:** Timer1/DDS в порядке; рывки из дыр STEP (цель=`mpg_cmd`, подача догоняет между детентами). Добавлен lead ≈ JOY lookahead за `mpg_cmd` на COAST; trim к cmd перед idle; stop без отката `dir_lock`.
- **Область:** motion_jog, MPG/РГИ.

### РГИ: idle не рвёт доезд; sticky dir после stop

- **Промпт:** скорость выросла, рывки остались; в логе `CRUISE`→`JOG stop`→новый старт, затем `DIR`/`BL`.
- **Результат:** stop только когда стоим на `mpg_cmd` и idle≥2.5 с; пока едем к cmd — сессию не гасим; любой тик (в т.ч. игнор реверса) продлевает idle; после stop `dir_lock` сохраняется, реверс только после N=3.
- **Область:** motion_jog, MPG/РГИ.

### РГИ: рабочая логика по скилу (одно cruise)

- **Промпт:** сверить со скилом и сделать рабочий код без прерывистого хода.
- **Результат:** цель=`mpg_cmd` без runway-реверса; на сессии реверс только игнор; idle 2с; planner/stepper не рвут живой cruise stop+start / сменой master.
- **Область:** motion_jog, planner, stepper_gen, MPG/РГИ.

### РГИ: реверс не abort’ит сессию (в т.ч. на цели)

- **Промпт:** прерывистость осталась; лог — новый `CRUISE`/pulse каждые сегменты, иногда DIR+/BL.
- **Результат:** при `mpg_active` обратные тики всегда discard; `dir_lock` липкий; смена направления только после idle-конца сессии.
- **Область:** motion_jog, MPG/РГИ.

### РГИ: на ходу не abort по реверсу

- **Промпт:** прерывистые движения; в логе 16×`rev ign` → новый разгон по кругу.
- **Результат:** пока `pos != target` обратные тики только отбрасываются (без halt); смена направления — когда цель достигнута. `DEBUG_LEVEL` → INFO (меньше тормоза Serial).
- **Область:** motion_jog, config, MPG/РГИ.

### РГИ: дребезг реверса не рвёт cruise

- **Промпт:** стало лучше, но не идеально; в логе цикл `CRUISE` → 5×`rev ign` → новый старт/разгон.
- **Результат:** `MPG_REV_IGNORE_TICKS=16` (дребезг при быстром кручении); при реальном реверсе — halt без старта в сторону шума в том же poll.
- **Область:** motion_jog, MPG/РГИ.

### РГИ: cruise без отката цели на idle

- **Промпт:** после отката runway всё ещё рывки; в логе `stop` + новый разгон между паузами руки.
- **Результат:** runway снова за `mpg_cmd` (~60 мс); idle не откатывает цель (нет DIR-/BL); stop только через 2 с на текущей pos; меньше Serial-спама на ext.
- **Область:** motion_jog, MPG/РГИ.

### РГИ: убран runaway за mpg_cmd (реверс/люфт)

- **Промпт:** после правки runway в логе спам `ext`, реверсы `DIR`, `BL START/ABORT`.
- **Результат:** цель снова не уходит за `mpg_cmd`; idle без спама retarget и без отката назад; `MPG_IDLE_DECEL_MS=600` сохранён (одно движение между детентами).
- **Область:** motion_jog, MPG/РГИ.

### РГИ: одно движение без рывков (runway + idle)

- **Промпт:** при быстром кручении РГИ мотор рывками; по скилу — одно движение без стопа между тиками.
- **Результат:** (частично) увеличен idle до 600 мс; попытка runway за `mpg_cmd` откатена — см. запись выше.
- **Область:** motion_jog, MPG/РГИ.
