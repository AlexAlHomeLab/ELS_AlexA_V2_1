# История изменений

Журнал изменений **функционала** прошивки ELS AlexA V2.1.  
Формат и правила: `.cursor/skills/els-changelog/SKILL.md`.

## 2026-07-14

### Divider: A/B/R живут при смене режима

- **Промпт:** при уходе в другие режимы и возврате не сбивать настройки и R; сброс угла только Select.
- **Результат:** `mode_divider_enter` больше не сбрасывает A/B/`zero_count`; Select только обновляет опорный ноль (B не трогает). Скил `els-divider`.
- **Область:** mode_divider, FSM.

### Divider: A минимум и default = 2

- **Промпт:** деление A=1 бессмысленно — начальное и минимальное число частей = 2.
- **Результат:** `DIVIDER_MIN_PARTS` / `DIVIDER_DEFAULT_PARTS` = 2; Down не опускает A ниже 2; скил `els-divider` обновлён.
- **Область:** mode_divider.

### LCD Xdia: квант 0.001 для радиуса и диаметра

- **Промпт:** по фактическим шагам отображать изменение 0.001 и на R, и на D.
- **Результат:** в `lcd_steps_to_parts` диаметр ×2 до деления + округление к 0.001 (раньше ×2 после trunc давало только 0.002). Скил `els-display-menu`.
- **Область:** LCD, Xdia.

### РГИ + Xdia: цена деления в диаметре

- **Промпт:** в режиме диаметра РГИ давал 0.01/0.1 мм радиуса; на LCD шаг удваивался.
- **Результат:** на оси X при Xdia=диаметр тик 0.01 → 0.005 мм, Rapid 0.1 → 0.05 мм радиуса (на экране 0.01/0.1 диаметра). x1step и Z без изменений. Скилы `els-rgi-mpg` / `els-display-menu`.
- **Область:** РГИ, Xdia, motion_jog.

### Меню UINT: clamp min/max во время правки

- **Промпт:** у Xdia можно поднять значение выше 1 при редактировании.
- **Результат:** после Up/Down для `PTYPE_UINT` значение сразу clamp в `min_v`/`max_v` (Xdia остаётся 0/1, CrdU ≤2).
- **Область:** ui_menu.

### Режим отображения X: радиус / диаметр (Xdia)

- **Промпт:** выбор R/D для координаты X; в диаметре LCD ×2; меню первым; default радиус + define.
- **Результат:** меню `Xdia` (0=R, 1=D), буква оси R/D вместо X; ×2 только мм/дюйм (и MPG по X); EEPROM display V2; `$47`; скил `els-display-menu`.
- **Область:** LCD, config_display, меню, Serial.

### Beep при защёлке подачи к лимиту

- **Промпт:** добавить beep на защёлку подачи между лимитами; обновить скил.
- **Результат:** при успешном `motion_jog_go_limit_latch` — короткий beep 40 мс (как клик лимита). Скилы `els-joy-feed` / `els-limits` обновлены.
- **Область:** joy latch, buzzer, motion_jog.

### MODE_OFF: debounce при переключении режимов

- **Промпт:** при повороте селектора кратко включается MODE_OFF — нужна задержка подтверждения.
- **Результат:** вход в MODE_OFF только после `MODE_OFF_DEBOUNCE_MS` (150 мс) непрерывного `scan==0`; выход сразу. Скил обновлён.
- **Область:** селектор режимов, ui_switches.

### MODE_OFF и soft-latch EN_X

- **Промпт:** при включении станка EN_X off; перед движением по X — enable и защёлка; MODE_OFF (все пины селектора) — EN_X off, блок joy/РГИ, экран CNC OFF + RPM + версия; скил селектора режимов.
- **Результат:** `EN_X_SOFT_LATCH` (D46 HIGH=on), `mode_off` в `ui_switches`, экран MODE_OFF на LCD, EN_Z только `#define` D45. E-Stop→EN не активен. Скил `els-mode-selector`.
- **Область:** селектор режимов, EN_X, jog/MPG, LCD.

### РГИ: отдельные скорости по режимам (compile-time)

- **Промпт:** отдельные настройки скорости для каждого режима РГИ (1:1, 0.01, 0.1 LIVE, точный подвод); вариант A — константы в config.
- **Результат:** в `config_mpg.h` — `MPG_SPEED_*` по оси X/Z для четырёх режимов; `mpg_speed_mm_min()` в `motion_jog.cpp` заменяет pot/rapid для всех вызовов planner с тегом MPG. Джойстик без изменений.
- **Область:** MPG, motion_jog, config.

### РГИ: режим Rapid «точный подвод» и маркер LCD

- **Промпт:** выбор функционала Rapid+РГИ через дефайн; режим точного подвода (цель при Rapid, движение при отпускании); маркер `>` на LCD (`MPG>X …`).
- **Результат:** `config_mpg.h` — `MPG_RAPID_MODE_LIVE` / `MPG_RAPID_MODE_APPROACH` (по умолчанию APPROACH). В APPROACH Rapid+РГИ наращивает `mpg_cmd` без движения; отпускание Rapid запускает подвод с подачей pot. LCD показывает `>` при нажатом Rapid.
- **Область:** MPG, motion_jog, LCD, config.
